# generate_bdio.py

Generates a Black Duck BDIO v2 (Bill of Distribution Input/Output) file from AOSP (Android Open Source Project) build artifacts and optionally uploads it to a Black Duck server. External repos that cannot be resolved to a known GitHub component can be signature-scanned using Black Duck Detect.

## Prerequisites

- **Python 3.6+**
- **AOSP build environment** with the following artifacts available:
  - `module-info.json` -- generated by the AOSP build system (typically found at `out/target/product/<device>/module-info.json`)
  - `repo-list.txt` -- output of `repo list` run from the AOSP source root
- **AOSP source tree** -- required for reading `METADATA` files from `external/` repos and for signature scanning
- **Black Duck server** -- required for BDIO upload and signature scanning
- **Black Duck API token** -- generated from the Black Duck UI under User > API Tokens
- **curl** -- required on the system PATH for signature scanning (used to download the Detect script)

## Configuration

Black Duck connection settings can be provided as command-line arguments or environment variables:

| Setting | Argument | Environment Variable |
|---|---|---|
| API token | `--bd-api-token` | `BLACKDUCK_API_TOKEN` |
| Server URL | `--bd-url` | `BLACKDUCK_URL` |
| Trust server certificate | `--bd-trust-cert` | `BLACKDUCK_TRUST_CERT` (`1`, `true`, or `yes`) |

Environment variables are used as defaults when the corresponding argument is not provided.

## Usage

### Required Arguments

```
--module-info PATH    Path to module-info.json from the AOSP build
--repo-list PATH      Path to repo-list.txt from 'repo list'
--android-version VER Android version string (e.g. android-16.0.0_r4)
--bd_project NAME     Black Duck project name
--bd_version VER      Black Duck project version
```

### Optional Arguments

```
--aosp-root PATH         Path to the AOSP source root (required for METADATA
                         parsing and signature scanning)
--output, -o PATH        Output BDIO file path (default: aosp-sbom.bdio)
--subfolder PATH         Only process repos/paths within this subfolder
                         (e.g. external)
--list-packages          List package identifiers and exit without generating BDIO
--sigscan-external       Run signature scan on external repos that could not be
                         resolved to a GitHub component
--bd-api-token TOKEN     Black Duck API token
--bd-url URL             Black Duck server URL
--bd-trust-cert          Trust the Black Duck server's SSL certificate
--debug                  Enable debug logging
```

### Basic Example

Generate a BDIO file without uploading:

```bash
python3 generate_bdio.py \
    --module-info out/target/product/generic_arm64/module-info.json \
    --repo-list repo-list.txt \
    --android-version android-16.0.0_r4 \
    --bd_project "AOSP" \
    --bd_version "16.0.0_r4" \
    --aosp-root /path/to/aosp \
    -o aosp-sbom.bdio
```

### Generate and Auto-Upload

When `--output` is omitted and Black Duck credentials are configured, the BDIO file is automatically uploaded after generation:

```bash
export BLACKDUCK_API_TOKEN="your-api-token"
export BLACKDUCK_URL="https://your-blackduck-server.com"
export BLACKDUCK_TRUST_CERT=true

python3 generate_bdio.py \
    --module-info out/target/product/generic_arm64/module-info.json \
    --repo-list repo-list.txt \
    --android-version android-16.0.0_r4 \
    --bd_project "AOSP" \
    --bd_version "16.0.0_r4" \
    --aosp-root /path/to/aosp
```

### Signature Scanning External Repos

External repos under `external/` that cannot be matched to a GitHub URL via their `METADATA` file are candidates for signature scanning. Use `--sigscan-external` to scan them with Black Duck Detect:

```bash
python3 generate_bdio.py \
    --module-info out/target/product/generic_arm64/module-info.json \
    --repo-list repo-list.txt \
    --android-version android-16.0.0_r4 \
    --bd_project "AOSP" \
    --bd_version "16.0.0_r4" \
    --aosp-root /path/to/aosp \
    --sigscan-external \
    --bd-api-token "your-api-token" \
    --bd-url "https://your-blackduck-server.com" \
    --bd-trust-cert
```

Signature scanning requires `--aosp-root`, `--bd-api-token`, and `--bd-url`.

Repos are scanned in place under `<aosp-root>/external/` in batches of up to 4 GB. A `.bdignore` file is temporarily written to the `external/` directory to exclude all folders except those in the current batch. Each batch produces a uniquely named codelocation on the Black Duck server (suffixed with the batch number).

### List Packages Only

To inspect which components would be included without generating a BDIO file:

```bash
python3 generate_bdio.py \
    --module-info out/target/product/generic_arm64/module-info.json \
    --repo-list repo-list.txt \
    --android-version android-16.0.0_r4 \
    --bd_project "AOSP" \
    --bd_version "16.0.0_r4" \
    --aosp-root /path/to/aosp \
    --list-packages
```

## How It Works

1. **Parse inputs** -- reads `repo-list.txt` and `module-info.json` to determine which repos and installed paths are relevant. Filters to `art/`, `bionic/`, `external/`, `frameworks/`, `hardware/`, `packages/`, `prebuilts/`, `system/`, and `tools/` prefixes, excluding test directories.

2. **Map paths to repos** -- matches each installed path to its owning repo using longest-prefix matching.

3. **Classify repos and build components**:
   - **External repos** (`external/` prefix): reads the `METADATA` file to extract a Git URL and version. If a GitHub URL is found, a GitHub-namespaced component is created. Otherwise the repo is marked for signature scanning.
   - **Platform repos** (everything else): creates an Android-namespaced platform component.

4. **Generate BDIO** -- assembles the BDIO v2 JSON-LD document and writes it as a ZIP file containing `bdio-header.jsonld` and `bdio-entry-00.jsonld`.

5. **Upload** -- if no explicit output path is given and Black Duck credentials are available, authenticates to the Black Duck API and uploads the BDIO file.

6. **Signature scan** -- if `--sigscan-external` is set, scans unresolved external repos using Black Duck Detect, batched to stay within the 4 GB size limit per scan.

## Generating Input Files

### repo-list.txt

From the AOSP source root:

```bash
repo list > repo-list.txt
```

### module-info.json

Build the AOSP target, then locate `module-info.json`:

```bash
source build/envsetup.sh
lunch <target>
make -j$(nproc)
```

The file is typically at `out/target/product/<device>/module-info.json`.
